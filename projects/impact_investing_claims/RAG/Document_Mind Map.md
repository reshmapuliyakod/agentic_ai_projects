# Cross-Document Verification Map

## The Core Idea
```
No single document tells the full story.
Greenwashing is found in the GAPS BETWEEN documents.
This is what makes UoP bond verification genuinely hard
and why a RAG system that treats documents in isolation will fail.
```

## Document Trust Hierarchy

```
MOST RELIABLE                                    LEAST RELIABLE
(Audited/Legal)                                  (Self-reported)
     │                                                  │
     ▼                                                  ▼
─────────────────────────────────────────────────────────────────
Annual Report     Final Terms    Allocation    Impact    Sust.
Financial Notes   (legal doc)    Report        Report    Report
                                 (if verified) 
     │                 │              │            │         │
  Audited           Legally        External     Often      Self-
  by Big 4          binding        assurance    unverified reported
                                   sometimes    
─────────────────────────────────────────────────────────────────

IMPLICATION FOR YOUR SYSTEM:
When documents contradict each other,
weight more reliable sources higher.
Flag contradictions between high-trust and low-trust docs
as higher risk than contradictions within low-trust docs.
```

## The Red Flag Severity Framework

```
Not all flags are equal. Your system needs to grade them.

SEVERITY 3: CRITICAL (Potential Fraud / Regulatory Breach)
────────────────────────────────────────────────────────────
  Allocated proceeds > Proceeds raised
  Funded projects explicitly on exclusion list
  Allocation Report never published (>2 years post issuance)
  Financial notes show no ring-fencing of proceeds
  Projects funded predate bond by >3 years with no explanation

SEVERITY 2: SIGNIFICANT (Material Greenwashing Risk)
────────────────────────────────────────────────────────────
  SPO reviewed different framework version than in use
  Impact methodology not disclosed
  Macro emissions increasing while impact claims are positive
  Capex data shows <10% going to green activities
  Cumulative reports don't reconcile
  No external verification of allocation or impact
  Reporting significantly delayed (>12 months)

SEVERITY 1: MODERATE (Amber Flag, Needs Scrutiny)
────────────────────────────────────────────────────────────
  Vague eligibility criteria (no measurable thresholds)
  Impact claims not pro-rated to bond's share of project
  Scope 3 emissions not disclosed
  SPO is >2 years old with no renewal
  Only aggregate allocation (no project-level detail)
  Qualitative claims dominate over quantitative

SEVERITY 0: INFORMATIONAL (Note but not a flag)
────────────────────────────────────────────────────────────
  Minor delays in reporting (<6 months)
  Framework updated (positive if SPO updated too)
  Partial allocation in year 1 (normal for large programmes)
  Different impact metrics year over year (methodology evolution)
```

## How This Maps to Your RAG System

```
PHASE 1 (Basic RAG)
  Focus:    Single document extraction
  Document: Allocation Report (most structured)
  Goal:     Can I extract the must-have fields accurately?

PHASE 2 (Hybrid Search)
  Focus:    Structured fields + unstructured claims
            from same document
  Document: Impact Report (has both)
  Goal:     Can I get the number AND the methodology claim?

PHASE 3 (Query Intelligence)
  Focus:    Cross-document questions
  Documents: Framework + Allocation Report
  Goal:     "Were all funded categories eligible?"
             Requires decomposition across two docs

PHASE 4 (Adaptive RAG)
  Focus:    System decides which documents to consult
  Question: "Is this bond credible?"
  Goal:     System identifies it needs
             Framework + SPO + Allocation + Impact
             not just one document

PHASE 5 (Self-Corrective RAG)
  Focus:    Contradiction detection
  Goal:     "Impact Report says X,
             Sustainability Report says Y,
             these are inconsistent → flag"

PHASE 6 (Agentic Router)
  Focus:    Full verification chain
  Goal:     Given issuer + bond ISIN →
             Pull all documents →
             Run all verification chains →
             Output severity-graded flag report
```
